var vs let vs const Declaracion de Variables

Ejemplo descriptivo


            
              
//ES5
(function() {
    console.log(x); // x no está definida aún.
    if(true) {
        var x = "hola mundo";
    }
    console.log(x); 
    // Imprime "hola mundo", porque "var" hace que sea global
    // a la función;
})();

//ES6
//LET
(function() {
    if(true) {
        let x = "hola mundo";
    }
    console.log(x); 
    //Da error, porque "x" ha sido definida dentro del "if"
})();

//CONST
(function() {
    const PI;
    PI = 3.15; 
    // ERROR, porque ha de asignarse un valor en la 
    // declaración
})();

(function() {
    const PI = 3.15;
    PI = 3.14159;
    // ERROR de nuevo, porque es sólo-lectura
})();

Descripción

Además de var, ahora tenemos acceso a dos nuevos identificadores para almacenar valores let y const. Cada una cuenta con sus propias características.

Características

  • A diferencia de las sentencias var, let y const no se elevan a la parte superior de su ámbito de inclusión.
  • Ahora con const podemos crear constantes que sólo se puedan leer y no modificar a lo largo del código.
  • Podemos declarar variables con let en lugar de var si no queremos que sean accesibles más allá de un ámbito.
